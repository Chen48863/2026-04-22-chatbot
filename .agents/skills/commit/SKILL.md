# Commit（提交推送）Skill

## 描述

此技能用於執行 Git 提交與推送操作。當使用者要求提交程式碼時，AI Agent 應依照以下流程完成 `git add`、`git commit` 與 `git push` 操作，確保程式碼變更被正確提交至遠端儲存庫。

## 觸發條件

當使用者提出以下類型的請求時啟用此技能：
- 「/commit」
- 「幫我提交程式碼」
- 「推送到遠端」
- 「git push」
- 「提交並推送」
- 「上傳程式碼」

## 執行流程

AI Agent 應按照以下步驟依序執行：

### 步驟 1：檢查 Git 狀態

```bash
git status
```

- 確認目前所在分支
- 確認是否有未追蹤或已修改的檔案
- 若無任何變更，回報「沒有需要提交的變更」並結束

### 步驟 2：設定 Git 使用者資訊

```bash
git config user.name "Antigravity"
git config user.email "Antigravity"
```

> 使用 `git config`（不加 `--global`），僅對當前儲存庫生效。

### 步驟 3：加入變更檔案

```bash
# 加入所有變更
git add .
```

或依使用者指定，僅加入特定檔案：

```bash
git add <檔案路徑>
```

### 步驟 4：提交變更

```bash
git commit -m "提交訊息"
```

#### Commit Message 撰寫規範

提交訊息應遵循以下格式：

```
<類型>: <簡短描述>
```

##### 類型（Type）定義：

| 類型 | 說明 | 範例 |
|------|------|------|
| `feat` | 新增功能 | `feat: 新增使用者登入功能` |
| `fix` | 修復錯誤 | `fix: 修復首頁載入失敗問題` |
| `docs` | 文件變更 | `docs: 更新 README 說明` |
| `style` | 程式碼格式調整（不影響功能） | `style: 統一縮排格式` |
| `refactor` | 程式碼重構（不影響功能） | `refactor: 重構資料庫查詢邏輯` |
| `test` | 新增或修改測試 | `test: 新增使用者 API 測試案例` |
| `chore` | 雜項變更（建構、工具等） | `chore: 更新依賴套件版本` |
| `init` | 專案初始化 | `init: 初始化專案結構` |

##### 撰寫原則：
- 使用**繁體中文**撰寫描述
- 簡短描述控制在 **50 字以內**
- 描述應說明「做了什麼」，而非「怎麼做」
- 若使用者未指定提交訊息，AI Agent 應根據變更內容**自動生成**適當的提交訊息

### 步驟 5：推送至遠端

```bash
git push origin <分支名稱>
```

- 預設推送至當前分支
- 若遠端尚無此分支，使用：

```bash
git push -u origin <分支名稱>
```

### 步驟 6：確認結果

推送完成後，回報以下資訊：

| 項目 | 內容 |
|------|------|
| **提交分支** | [分支名稱] |
| **提交雜湊** | [commit hash 前 7 碼] |
| **提交訊息** | [commit message] |
| **變更檔案數** | [數量] |
| **新增行數** | [數量] |
| **刪除行數** | [數量] |
| **推送狀態** | 成功 / 失敗 |

## 錯誤處理

### 常見錯誤與處理方式

| 錯誤情境 | 處理方式 |
|----------|----------|
| 尚未初始化 Git 儲存庫 | 詢問使用者是否執行 `git init` |
| 沒有設定遠端儲存庫 | 提示使用者執行 `git remote add origin <URL>` |
| 推送被拒絕（遠端有新提交） | 建議使用者先執行 `git pull --rebase` |
| 合併衝突 | 列出衝突檔案，提示使用者手動解決 |
| 認證失敗 | 提示使用者檢查 Git 認證設定（SSH Key / Token） |
| 沒有任何變更可提交 | 回報「工作區域是乾淨的，沒有需要提交的變更」 |

## 完整執行範例

```bash
# 1. 檢查狀態
git status

# 2. 設定使用者資訊
git config user.name "Antigravity"
git config user.email "Antigravity"

# 3. 加入所有變更
git add .

# 4. 提交（自動生成提交訊息）
git commit -m "feat: 新增使用者管理功能"

# 5. 推送至遠端
git push origin main

# 6. 確認結果
git log -1 --oneline
```

## 注意事項

- 提交前應確認**不要提交敏感資訊**（如 API Key、密碼、`.env` 檔案）
- 若專案無 `.gitignore` 檔案，應建議使用者建立
- 提交訊息應以**繁體中文**撰寫
- 每次提交應為**一個完整的邏輯變更**，避免將不相關的變更混在同一次提交
- 推送前應確認目標分支是否正確，避免誤推至 `main` / `master`
- 若使用者未指定提交訊息，AI Agent 應根據 `git diff --staged` 的內容自動產生有意義的提交訊息
