# PRD (Product Requirements Document) Skill

## 描述

此技能用於生成產品需求文件（PRD）。當使用者要求撰寫 PRD 時，AI Agent 應依照以下結構產出完整的產品需求文件。

## 觸發條件

當使用者提出以下類型的請求時啟用此技能：
- 「幫我寫一份 PRD」
- 「產生產品需求文件」
- 「建立新功能的需求規格」

## 輸出格式

請依照以下模板撰寫 PRD：

### 1. 概述（Overview）
- **產品名稱**：[名稱]
- **版本**：[版本號]
- **日期**：[撰寫日期]
- **作者**：[作者名稱]

### 2. 目標與背景（Goals & Background）
- 描述產品或功能的目標
- 說明為什麼需要這個產品/功能
- 目前的痛點與需求

### 3. 使用者故事（User Stories）
- 以「作為 [角色]，我希望 [功能]，以便 [目的]」格式撰寫
- 列出主要使用者場景

### 4. 功能需求（Functional Requirements）
- 詳細列出所有功能需求
- 每項需求需包含優先級（P0/P1/P2）
- 需求的驗收標準（Acceptance Criteria）

### 5. 非功能需求（Non-Functional Requirements）
- 效能需求
- 安全性需求
- 可擴展性需求
- 相容性需求

### 6. 系統架構（System Architecture）
- 技術架構概述
- 系統元件與其互動方式
- 資料流程圖

### 7. 時程規劃（Timeline）
- 各階段的里程碑
- 預估完成日期

### 8. 風險評估（Risk Assessment）
- 潛在風險與其影響
- 風險緩解策略

### 9. 成功指標（Success Metrics）
- 衡量產品成功的關鍵指標（KPI）
- 預期目標數值

## 注意事項

- 所有 PRD 內容應以**繁體中文**撰寫
- 需求描述應具體、可衡量
- 優先級劃分需明確（P0 = 必要、P1 = 重要、P2 = 可選）
- 如有不確定的需求，應標註為「待確認」
