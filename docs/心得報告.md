## Q1. 你設計的哪一個 Skill 效果最好？為什麼？哪一個效果最差？你認為原因是什麼？

在本次專案中，我總共設計了六個 Skill：PRD、Architecture、Models、Implement、Test、Commit。

**效果最好的 Skill：Implement（程式碼實作）**

Implement Skill 的效果最為顯著，因為它能夠根據前面產出的 PRD、Architecture、Models 文件，一次性生成完整且可執行的 FastAPI 應用程式，包含後端路由、資料庫模型、前端模板、CSS 樣式與 JavaScript 互動邏輯。生成的程式碼結構清晰，透過 `python app.py` 即可直接啟動運作，大幅縮短了開發時間。我認為效果好的原因在於，Implement Skill 的指令明確定義了技術架構（FastAPI + SQLite + Jinja2），並且有前面三份文件作為完整的需求脈絡，讓 AI 能精準理解要實作什麼功能。

**效果最差的 Skill：PRD（產品需求文件）**

PRD Skill 在初期使用時遇到了「No matching results」的觸發問題，無法被正確啟用。我認為原因是 Skill 的觸發條件設定不夠明確，導致 AI Agent 無法正確匹配使用者的輸入意圖。後來經過調整觸發關鍵字與描述後才順利運作。這讓我理解到 Skill 的設計不只是內容本身，觸發條件的精準度同樣至關重要。

## Q2. 在用 AI 產生程式碼的過程中，你遇到什麼問題是 AI 沒辦法自己解決、需要你介入處理的？

在使用 AI 產生程式碼的過程中，我遇到了以下幾個需要人為介入的問題：

1. **中文路徑編碼問題**：我的專案資料夾路徑包含中文字元（「生成式AI應用：AI agent 開發與實作(創能綜班)[0771]」），導致在終端機執行 `pip install -r requirements.txt` 和 `python app.py` 時出現找不到檔案的錯誤。AI 雖然能嘗試改用完整路徑或直接指定套件名稱來解決，但根本原因是作業系統對中文路徑的處理差異，這需要我自己理解並確認解決方案是否可行。

2. **GitHub 認證與儲存庫建立**：AI 無法代替我登入 GitHub 建立新的 Repository，也無法處理 Git 的身份認證（如 SSH Key 或 Personal Access Token 的設定）。這些操作必須由我手動在瀏覽器和本機環境中完成。

3. **個人化內容的判斷**：像是心得報告、姓名學號等個人資訊，以及對 Skill 效果好壞的主觀評價，AI 只能根據開發過程的客觀紀錄來推測，最終仍需要我本人確認內容是否符合實際感受。

整體而言，AI 在「產出結構化內容」和「執行重複性指令」方面表現非常出色，但在「環境配置差異」、「第三方服務認證」和「主觀判斷」這些面向，仍然需要開發者本人介入處理。
